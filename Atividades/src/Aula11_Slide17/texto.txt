<LISTA 01>
1. O que é Visualização? O que é Visualização Científica? Qual a diferença entre Visualização Científica e Visualização de Informação?
R: Visualização é a comunicação da informação através de representações gráficas. Visualização científica é uma área dedicada à geração de imagens que auxiliam seus usuários na compreensão de dados e processos. Visualização de Informações é uma área de aplicação de técnicas de computação gráfica, geralmente interativas, visando auxiliar o processo de análise e compreensão de um conjunto de dados, através de representações gráficas manipuláveis.

2. Qual a relação da Visualização de Informação com a área de Mineração de Dados?
R: A mineração de dados, têm como objetivo extrair e analisar dados, é nesse último quesito que a visualização de informação entra, isso porque, com a visualização de informações, pode-se transmitir as informações mais rápida e claramente, sendo que podem-se expor os padrões encontrados de forma gráfica, em um modelo onde fique mais intuitivo entender e encontrar informações do que apenas com os dados minerados sem processamento ou visualização.

3. O que são scatterplots? Como eles são criados e como podem ser usados? Qual a principal desvantagem dos scatterplots e como ela pode ser superada?
R: Scatterplots é um tipo de visualização onde os dados são representados em um gráfico no plano cartesiano, através das coordenadas x e y. Portanto, pode ser comparado dois
atributos mapeando cada instância dos dados no plano. É uma ótima forma de visualizar a correlação de duas ou mais medidas ao mesmo tempo
-Desvantagens:  pode ser difícil de ser entendida por um usuário inexperiente,
pois contém valor de medida nos dois eixos, e a terceira medida, que é opcional,
contribui para a complexidade da interpretação.
-Usa rótulos descritivos é uma boa forma de facilitar a interpretação da visualização.
-Os valores podem ser colocados uns sobre os outros e não serão visíveis até que você aplique zoom.

4. Qual a importância do pré-processamento dos dados? Cite e explique dois métodos de pré-processamento de dados.
R: Na maioria dos casos, as bases de dados utilizadas contém informações incompletas(dados faltando), unidades de medidas diferentes, e alguns casos até dados errados. Então, para um melhor aproveitamento é necessário fazer um tratamento nos dados, para que fiquem concisos, tal técnica é chamada de pré-processamento dos dados. Um possível método para tratamento de dados faltantes é o de assinalar um valor sentinela, utilizando a média dos valores, ou a vizinhança mais próxima. Outro método utilizado nas situações em que a dimensionalidade excede a capacidade da técnica de visualização é o de redução, preservando o máximo possível de informação contida nos dados. Essa redução pode ser feita manualmente, selecionando os atributos, ou usando alguma técnica computacional.

5. Quais são as três estratégias mais comuns para lidar com valores faltando nos dados? Comente as vantagens e desvantagens de cada uma dessas estratégias.
R: * Descartar as instâncias com defeito:
pode representar grande perda de informação;
mais fácil;
     * Assinalar um valor sentinela:
    cuidado para não usar a sentinela nos cálculos;
     * Calcular um valor substituto:
    Assinalar o valor médio : afeta minimamente as estatísticas da variável, mas pode 
obscurecer valores espúrios;
Assinalar um valor baseado no vizinho mais próximo (mais adequado).

6. Qual a importância da interação na visualização de dados?
R: É importante, devido ao fato de que, com ela, é possível mostrar porções dos dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe, você também pode combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Ou seja, a interação pode contribuir para mostrar informações referentes a cada parte de uma visualização, também pode mostrar a influência de uma região dela em outra, se bem produzida pode até ser determinante na compreensão de uma visualização. 

7) A taxonomia de Keim classifica as técnicas de interação e distorção em cinco categorias. Cite e explique duas dessas categorias.
R:     1a. Projeção Dinâmica - Dinamicamente alterar as projeções de modo a explorar um conjunto de dados multidimensional.). Exemplos: XGobi e Grand Tour.
2b. Filtros interativos - Ao explorar grandes conjuntos de dados, é importante interativamente particionar o conjunto de dados em segmentos e se concentrar em subgrupos
interessantes. Seleção direta ou por especificação de propriedades. Exemplo: Magic Lenses, InfoCrystal e dynamic queries
3c. Zoom interativo - Uma vez que a quantidade máxima de informação pode ser limitada pela resolução de uma tela, o zoom é uma técnica fundamental para superar esta
limitação. Exemplos: TableLens e IVEE/Spotfire.

8. Qual o objetivo do algoritmo Marching Squares e para que tipo de dados ele pode ser aplicado? Explique em detalhes o algoritmo.
R: O algoritmo Marching Squares visa desenhar linhas entre os valores interpolados ao longo das bordas de uma célula, considerando-se dados sobre o peso dos vértices e um
valor de referência (isovalor).
Vértices de uma célula são anotados com:
    ‘1’ para valores iguais ou maiores que o isovalor;
    ‘0’, caso contrário.
Aplique uma interpolação linear ao longo das bordas da célula para calcular a posição exata do contorno da isolinha.
Recomenda-se utilizar para dados 2D, quando se quer exibir um limite,como por exemplo elevação de temperatura.

 9. Quais são as três categorias de técnicas para visualização volumétrica?
Explique o funcionamento geral das técnicas de cada uma dessas categorias. Quais as vantagens e desvantagens do uso de técnicas de cada uma dessas categorias?
R:    
Tecnicas de fatiamento (slicing): dados são capturados considerando um
plano de recorte 2D alinhado com um dos eixos.
Planos de recorte são usados reduzindo os dados de 3D para 2D
Os dados são recortados com um plano dada uma direção e posição.
Em uma implementação simples a normal do plano coincide com um dos eixos dos dados.
Animação pode ser usada para a visualização de associações entre fatias subsequentes
Técnicas de isosuperfície: gera uma superfície dado um valor e usa técnicas de visualização explícita.
Outra forma de representar dados 3D é por meio da extração de isosuperfícies.
O algoritmo mais conhecido para extração é conhecido como Marching Cubes.
Grande consumo de memória para armazenar o volume resultante
Um voxel é definido pelos valores de seus 8 cantos.
A contribuição do voxel para compor a superfície depende se esses valores são maiores ou menores que um isovalor
Para orientações arbitrárias, cada voxel intersectado pode influenciar o valor de um ou mais pixels no plano.
O pixel resultante pode ser derivado fazendo a re-amostragem dos dados, considerando o voxel mais próximo do plano.    
Rendering volumétrico direto: calcula a cor dos pixels lançando raios no
volume; ou projeta o volume em um plano e usa alguma estratégia para acumular efeitos nos pixels
não são criados polígonos tridimensionais, mas os pixels na imagem resultante são computados individualmente
Mapeamento direto: projeta cada voxel no plano de projeção e determina quais pixels são afetados e de que forma.
Mapeamento inverso(ray casting): lança um raio para cada pixel no plano de projeção através do volume, amostrando valores ao longo desse determinando o valor
de cada pixel.

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

11. Data a técnica para dados multivariados chamada Coordenadas Paralelas:
Explique o layout e funcionamento dessa técnica.
R: Nas coordenadas paralelas, os eixos são paralelos e espaçados uniformemente, e as instâncias de dados são polilinhas que cruzam esses eixos em uma posição proporcional ao valor na dimensão.
Na interpretação desse tipo de visualização podemos:
● Localizar grupos de polilinhas semelhantes (indicam correlação parcial entre pares de dimensões).
● Localizar pontos de cruzamento (indicam correlação parcial negativa).
● Localizar polilinhas diferentes (indicam outliers).

A técnica de Coordenadas Paralelas consiste em um gráfico onde no eixo X estão os atributos e no eixo Y estão o intervalo desses atributos.

b. Cite duas desvantagens dessa técnica e modos pelos quais essas desvantagens podem ser superadas.
R: Interpretar relações entre dimensões não consecutivas pode ser difícil, Interação pode ajudar a sobrepor esse problema indicando relações que se estendem por todas as dimensões.
Grandes grupos sobrepostos: Semi-transparência.

<LISTA 02>
1.  A Treemap é uma técnica clássica utilizada para visualização de dados hierárquicos.
Explique o layout e funcionamento dessa técnica
R:  Tree Mapping é um método para exibir dados hierárquicos usando retângulos aninhados, exibindo dados hierárquicos (árvore-estruturada) como um conjunto de retângulos aninhados. Para cada ramo da árvore é definido um retângulo, que é, então, preenchido com retângulos menores lado a lado, os quais representam sub-ramos. Cada retângulo tem uma área proporcional a uma dimensão especificada nos dados.

2. Explique o funcionamento e layout de uma das técnicas ensinadas em aula para visualização de
grafos e redes arbitrárias.
R: Grafos de Ligação de Nós: Método baseado em força. Usam uma analogia de molas para representar as
arestas, com as posições dos nós iterativamente refinadas até se atingir um estado de equilíbrio com
o stress do sistema minimizado. Em cada iteração as forças sobre cada nó são calculadas, suas posições atualizadas e suas forças recalculadas.

3. Qual a importância da interação em Visualização da Informação?
R:  Possibilitar e facilitar que pessoas  manipulem a representação de dados  de modo que,  busque melhor compreender o conjunto de dados ali representado. Podendo facilitar o entendimento de padrões em regiões de interesse a serem destacados.

<AUX>
O pipeline de visualização apresenta diferentes estágios:
Modelagem dos dados; Seleção dos dados; Mapeamento visual dos dados; Definição dos parâmetros de cena; Geração da visualização.

Scatterplots
Uma das visualizações mais antigas desenvolvidas.Dois atributos são comparados mapeando cada instância de dados em um gráfico, podendo outros serem mapeados para cor e tamanho desse.
A visualização pode ser executada para diferentes fins:
Exploração: usuário tem os dados e quer verificar certas características.
Confirmação: existe uma hipótese sobre alguma característica e o usuário quer confirmá-la.
Apresentação: apresentar algum conceito ou conjunto de fatos a um público.


Uma variável pode ser:
Independente (ivi): seu valor não é afetado ou controlado por outra variável.
Exemplo: o tempo em uma série temporal.
Dependente (dvj): é afetada pela variação de uma ou mais variáveis independentes
Exemplo: a temperatura de uma região pode ser dependente da data, horário ou localização.

Oito variáveis visuais:
1 posição; 2 forma (ou marcador); 3 tamanho; 4 brilho; 5 cor; 6 orientação; 7 textura; 8 movimento.

Taxonomia
Uma taxonomia é um meio para transmitir uma classificação. Frequentemente hierárquica, uma taxonomia agrupa objetos similares e define relacionamentos. Ex: uni-dimensional linear; mapa bidimensional; mundo tridimensional; temporal; multidimensional; árvore; rede.

Distorção interativa – Mostrar porções os dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe.
Exemplos: distorções hiperbólicas e distorções esféricas (fisheye lens)

Linking e brushing interativo – Combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Mudanças interativas feitas em uma visualização são automaticamente refletidas nas outras visualizações. Exemplos: Polaris and Scalable Framework.

Independente (ivi): seu valor não é afetado ou controlado por outra variável.
Exemplo: o tempo em uma série temporal.
Dependente (dvj): é afetada pela variação de uma ou mais variáveis independentes
Exemplo: a temperatura de uma região pode ser dependente da data, horário ou localização.

Oito variáveis visuais:
1 posição; 2 forma (ou marcador); 3 tamanho; 4 brilho; 5 cor; 6 orientação; 7 textura; 8 movimento.

Taxonomia
Uma taxonomia é um meio para transmitir uma classificação. Frequentemente hierárquica, uma taxonomia agrupa objetos similares e define relacionamentos. Ex: uni-dimensional linear; mapa bidimensional; mundo tridimensional; temporal; multidimensional; árvore; rede.

Distorção interativa – Mostrar porções os dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe.
Exemplos: distorções hiperbólicas e distorções esféricas (fisheye lens)

Linking e brushing interativo – Combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Mudanças interativas feitas em uma visualização são automaticamente refletidas nas outras visualizações. Exemplos: Polaris and Scalable Framework.

1. O que é Visualização? O que é Visualização Científica? Qual a diferença entre Visualização Científica e Visualização de Informação?
R: Visualização é a comunicação da informação através de representações gráficas. Visualização científica é uma área dedicada à geração de imagens que auxiliam seus usuários na compreensão de dados e processos. Visualização de Informações é uma área de aplicação de técnicas de computação gráfica, geralmente interativas, visando auxiliar o processo de análise e compreensão de um conjunto de dados, através de representações gráficas manipuláveis.

2. Qual a relação da Visualização de Informação com a área de Mineração de Dados?
R: A mineração de dados, têm como objetivo extrair e analisar dados, é nesse último quesito que a visualização de informação entra, isso porque, com a visualização de informações, pode-se transmitir as informações mais rápida e claramente, sendo que podem-se expor os padrões encontrados de forma gráfica, em um modelo onde fique mais intuitivo entender e encontrar informações do que apenas com os dados minerados sem processamento ou visualização.

3. O que são scatterplots? Como eles são criados e como podem ser usados? Qual a principal desvantagem dos scatterplots e como ela pode ser superada?
R: Scatterplots é um tipo de visualização onde os dados são representados em um gráfico no plano cartesiano, através das coordenadas x e y. Portanto, pode ser comparado dois
atributos mapeando cada instância dos dados no plano. É uma ótima forma de visualizar a correlação de duas ou mais medidas ao mesmo tempo
-Desvantagens:  pode ser difícil de ser entendida por um usuário inexperiente,
pois contém valor de medida nos dois eixos, e a terceira medida, que é opcional,
contribui para a complexidade da interpretação.
-Usa rótulos descritivos é uma boa forma de facilitar a interpretação da visualização.
-Os valores podem ser colocados uns sobre os outros e não serão visíveis até que você aplique zoom.

4. Qual a importância do pré-processamento dos dados? Cite e explique dois métodos de pré-processamento de dados.
R: Na maioria dos casos, as bases de dados utilizadas contém informações incompletas(dados faltando), unidades de medidas diferentes, e alguns casos até dados errados. Então, para um melhor aproveitamento é necessário fazer um tratamento nos dados, para que fiquem concisos, tal técnica é chamada de pré-processamento dos dados. Um possível método para tratamento de dados faltantes é o de assinalar um valor sentinela, utilizando a média dos valores, ou a vizinhança mais próxima. Outro método utilizado nas situações em que a dimensionalidade excede a capacidade da técnica de visualização é o de redução, preservando o máximo possível de informação contida nos dados. Essa redução pode ser feita manualmente, selecionando os atributos, ou usando alguma técnica computacional.

5. Quais são as três estratégias mais comuns para lidar com valores faltando nos dados? Comente as vantagens e desvantagens de cada uma dessas estratégias.
R: * Descartar as instâncias com defeito:
pode representar grande perda de informação;
mais fácil;
     * Assinalar um valor sentinela:
    cuidado para não usar a sentinela nos cálculos;
     * Calcular um valor substituto:
    Assinalar o valor médio : afeta minimamente as estatísticas da variável, mas pode 
obscurecer valores espúrios;
Assinalar um valor baseado no vizinho mais próximo (mais adequado).

6. Qual a importância da interação na visualização de dados?
R: É importante, devido ao fato de que, com ela, é possível mostrar porções dos dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe, você também pode combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Ou seja, a interação pode contribuir para mostrar informações referentes a cada parte de uma visualização, também pode mostrar a influência de uma região dela em outra, se bem produzida pode até ser determinante na compreensão de uma visualização. 

7) A taxonomia de Keim classifica as técnicas de interação e distorção em cinco categorias. Cite e explique duas dessas categorias.
R:     1a. Projeção Dinâmica - Dinamicamente alterar as projeções de modo a explorar um conjunto de dados multidimensional.). Exemplos: XGobi e Grand Tour.
2b. Filtros interativos - Ao explorar grandes conjuntos de dados, é importante interativamente particionar o conjunto de dados em segmentos e se concentrar em subgrupos
interessantes. Seleção direta ou por especificação de propriedades. Exemplo: Magic Lenses, InfoCrystal e dynamic queries
3c. Zoom interativo - Uma vez que a quantidade máxima de informação pode ser limitada pela resolução de uma tela, o zoom é uma técnica fundamental para superar esta
limitação. Exemplos: TableLens e IVEE/Spotfire.

8. Qual o objetivo do algoritmo Marching Squares e para que tipo de dados ele pode ser aplicado? Explique em detalhes o algoritmo.
R: O algoritmo Marching Squares visa desenhar linhas entre os valores interpolados ao longo das bordas de uma célula, considerando-se dados sobre o peso dos vértices e um
valor de referência (isovalor).
Vértices de uma célula são anotados com:
    ‘1’ para valores iguais ou maiores que o isovalor;
    ‘0’, caso contrário.
Aplique uma interpolação linear ao longo das bordas da célula para calcular a posição exata do contorno da isolinha.
Recomenda-se utilizar para dados 2D, quando se quer exibir um limite,como por exemplo elevação de temperatura.

 9. Quais são as três categorias de técnicas para visualização volumétrica?
Explique o funcionamento geral das técnicas de cada uma dessas categorias. Quais as vantagens e desvantagens do uso de técnicas de cada uma dessas categorias?
R:    
Tecnicas de fatiamento (slicing): dados são capturados considerando um
plano de recorte 2D alinhado com um dos eixos.
Planos de recorte são usados reduzindo os dados de 3D para 2D
Os dados são recortados com um plano dada uma direção e posição.
Em uma implementação simples a normal do plano coincide com um dos eixos dos dados.
Animação pode ser usada para a visualização de associações entre fatias subsequentes
Técnicas de isosuperfície: gera uma superfície dado um valor e usa técnicas de visualização explícita.
Outra forma de representar dados 3D é por meio da extração de isosuperfícies.
O algoritmo mais conhecido para extração é conhecido como Marching Cubes.
Grande consumo de memória para armazenar o volume resultante
Um voxel é definido pelos valores de seus 8 cantos.
A contribuição do voxel para compor a superfície depende se esses valores são maiores ou menores que um isovalor
Para orientações arbitrárias, cada voxel intersectado pode influenciar o valor de um ou mais pixels no plano.
O pixel resultante pode ser derivado fazendo a re-amostragem dos dados, considerando o voxel mais próximo do plano.    
Rendering volumétrico direto: calcula a cor dos pixels lançando raios no
volume; ou projeta o volume em um plano e usa alguma estratégia para acumular efeitos nos pixels
não são criados polígonos tridimensionais, mas os pixels na imagem resultante são computados individualmente
Mapeamento direto: projeta cada voxel no plano de projeção e determina quais pixels são afetados e de que forma.
Mapeamento inverso(ray casting): lança um raio para cada pixel no plano de projeção através do volume, amostrando valores ao longo desse determinando o valor
de cada pixel.

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

11. Data a técnica para dados multivariados chamada Coordenadas Paralelas:
Explique o layout e funcionamento dessa técnica.
R: Nas coordenadas paralelas, os eixos são paralelos e espaçados uniformemente, e as instâncias de dados são polilinhas que cruzam esses eixos em uma posição proporcional ao valor na dimensão.
Na interpretação desse tipo de visualização podemos:
● Localizar grupos de polilinhas semelhantes (indicam correlação parcial entre pares de dimensões).
● Localizar pontos de cruzamento (indicam correlação parcial negativa).
● Localizar polilinhas diferentes (indicam outliers).

A técnica de Coordenadas Paralelas consiste em um gráfico onde no eixo X estão os atributos e no eixo Y estão o intervalo desses atributos.

b. Cite duas desvantagens dessa técnica e modos pelos quais essas desvantagens podem ser superadas.
R: Interpretar relações entre dimensões não consecutivas pode ser difícil, Interação pode ajudar a sobrepor esse problema indicando relações que se estendem por todas as dimensões.
Grandes grupos sobrepostos: Semi-transparência.

arestas, com as posições dos nós iterativamente refinadas até se atingir um estado de equilíbrio com
o stress do sistema minimizado. Em cada iteração as forças sobre cada nó são calculadas, suas posições atualizadas e suas forças recalculadas.

R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

interessantes. Seleção direta ou por especificação de propriedades. Exemplo: Magic Lenses, InfoCrystal e dynamic queries
3c. Zoom interativo - Uma vez que a quantidade máxima de informação pode ser limitada pela resolução de uma tela, o zoom é uma técnica fundamental para superar esta
limitação. Exemplos: TableLens e IVEE/Spotfire.

8. Qual o objetivo do algoritmo Marching Squares e para que tipo de dados ele pode ser aplicado? Explique em detalhes o algoritmo.
R: O algoritmo Marching Squares visa desenhar linhas entre os valores interpolados ao longo das bordas de uma célula, considerando-se dados sobre o peso dos vértices e um
valor de referência (isovalor).
Vértices de uma célula são anotados com:
    ‘1’ para valores iguais ou maiores que o isovalor;
    ‘0’, caso contrário.
Aplique uma interpolação linear ao longo das bordas da célula para calcular a posição exata do contorno da isolinha.
Recomenda-se utilizar para dados 2D, quando se quer exibir um limite,como por exemplo elevação de temperatura.

R:

Os dados são recortados com um plano dada uma direção e posição.
Em uma implementação simples a normal do plano coincide com um dos eixos dos dados.
Animação pode ser usada para a visualização de associações entre fatias subsequentes

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

● Localizar polilinhas diferentes (indicam outliers).

R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

8. Qual o objetivo do algoritmo Marching Squares e para que tipo de dados ele pode ser aplicado? Explique em detalhes o algoritmo.
R: O algoritmo Marching Squares visa desenhar linhas entre os valores interpolados ao longo das bordas de uma célula, considerando-se dados sobre o peso dos vértices e um
valor de referência (isovalor).
Vértices de uma célula são anotados com:
    ‘1’ para valores iguais ou maiores que o isovalor;
    ‘0’, caso contrário.
Aplique uma interpolação linear ao longo das bordas da célula para calcular a posição exata do contorno da isolinha.
Recomenda-se utilizar para dados 2D, quando se quer exibir um limite,como por exemplo elevação de temperatura.

A contribuição do voxel para compor a superfície depende se esses valores são maiores ou menores que um isovalor
Para orientações arbitrárias, cada voxel intersectado pode influenciar o valor de um ou mais pixels no plano.
O pixel resultante pode ser derivado fazendo a re-amostragem dos dados, considerando o voxel mais próximo do plano.

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

Na interpretação desse tipo de visualização podemos:
● Localizar grupos de polilinhas semelhantes (indicam correlação parcial entre pares de dimensões).
● Localizar pontos de cruzamento (indicam correlação parcial negativa).
● Localizar polilinhas diferentes (indicam outliers).

A técnica de Coordenadas Paralelas consiste em um gráfico onde no eixo X estão os atributos e no eixo Y estão o intervalo desses atributos.

Grandes grupos sobrepostos: Semi-transparência.

Visualização científica é uma área dedicada à geração de imagens que auxiliam seus usuários na compreensão de dados e processos. Visualização de Informações é uma área de aplicação de técnicas de computação gráfica, geralmente interativas, visando auxiliar o processo de análise e compreensão de um conjunto de dados, através de representações gráficas manipuláveis.

2. Qual a relação da Visualização de Informação com a área de Mineração de Dados?
R: A mineração de dados, têm como objetivo extrair e analisar dados, é nesse último quesito que a visualização de informação entra, isso porque, com a visualização de informações, pode-se transmitir as informações mais rápida e claramente, sendo que podem-se expor os padrões encontrados de forma gráfica, em um modelo onde fique mais intuitivo entender e encontrar informações do que apenas com os dados minerados sem processamento ou visualização.

3. O que são scatterplots? Como eles são criados e como podem ser usados? Qual a principal desvantagem dos scatterplots e como ela pode ser superada?
R: Scatterplots é um tipo de visualização onde os dados são representados em um gráfico no plano cartesiano, através das coordenadas x e y. Portanto, pode ser comparado dois
atributos mapeando cada instância dos dados no plano. É uma ótima forma de visualizar a correlação de duas ou mais medidas ao mesmo tempo
-Desvantagens:  pode ser difícil de ser entendida por um usuário inexperiente,
pois contém valor de medida nos dois eixos, e a terceira medida, que é opcional,
contribui para a complexidade da interpretação.
-Usa rótulos descritivos é uma boa forma de facilitar a interpretação da visualização.
-Os valores podem ser colocados uns sobre os outros e não serão visíveis até que você aplique zoom.

4. Qual a importância do pré-processamento dos dados? Cite e explique dois métodos de pré-processamento de dados.
R: Na maioria dos casos, as bases de dados utilizadas contém informações incompletas(dados faltando), unidades de medidas diferentes, e alguns casos até dados errados. Então, para um melhor aproveitamento é necessário fazer um tratamento nos dados, para que fiquem concisos, tal técnica é chamada de pré-processamento dos dados. Um possível método para tratamento de dados faltantes é o de assinalar um valor sentinela, utilizando a média dos valores, ou a vizinhança mais próxima. Outro método utilizado nas situações em que a dimensionalidade excede a capacidade da técnica de visualização é o de redução, preservando o máximo possível de informação contida nos dados. Essa redução pode ser feita manualmente, selecionando os atributos, ou usando alguma técnica computacional.

6. Qual a importância da interação na visualização de dados?

Ou seja, a interação pode contribuir para mostrar informações referentes a cada parte de uma visualização, também pode mostrar a influência de uma região dela em outra, se bem produzida pode até ser determinante na compreensão de uma visualização. 

7) A taxonomia de Keim classifica as técnicas de interação e distorção em cinco categorias. Cite e explique duas dessas categorias.
R:     1a. Projeção Dinâmica - Dinamicamente alterar as projeções de modo a explorar um conjunto de dados multidimensional.). Exemplos: XGobi e Grand Tour.
2b. Filtros interativos - Ao explorar grandes conjuntos de dados, é importante interativamente particionar o conjunto de dados em segmentos e se concentrar em subgrupos
interessantes. Seleção direta ou por especificação de propriedades. Exemplo: Magic Lenses, InfoCrystal e dynamic queries
3c. Zoom interativo - Uma vez que a quantidade máxima de informação pode ser limitada pela resolução de uma tela, o zoom é uma técnica fundamental para superar esta
limitação. Exemplos: TableLens e IVEE/Spotfire.

8. Qual o objetivo do algoritmo Marching Squares e para que tipo de dados ele pode ser aplicado? Explique em detalhes o algoritmo.
R: O algoritmo Marching Squares visa desenhar linhas entre os valores interpolados ao longo das bordas de uma célula, considerando-se dados sobre o peso dos vértices e um
valor de referência (isovalor).
Vértices de uma célula são anotados com:
    ‘1’ para valores iguais ou maiores que o isovalor;
    ‘0’, caso contrário.
Aplique uma interpolação linear ao longo das bordas da célula para calcular a posição exata do contorno da isolinha.
Recomenda-se utilizar para dados 2D, quando se quer exibir um limite,como por exemplo elevação de temperatura.

 9. Quais são as três categorias de técnicas para visualização volumétrica?
Explique o funcionamento geral das técnicas de cada uma dessas categorias. Quais as vantagens e desvantagens do uso de técnicas de cada uma dessas categorias?
R:    
Tecnicas de fatiamento (slicing): dados são capturados considerando um
plano de recorte 2D alinhado com um dos eixos.
Planos de recorte são usados reduzindo os dados de 3D para 2D
Os dados são recortados com um plano dada uma direção e posição.
Em uma implementação simples a normal do plano coincide com um dos eixos dos dados.
Animação pode ser usada para a visualização de associações entre fatias subsequentes
Técnicas de isosuperfície: gera uma superfície dado um valor e usa técnicas de visualização explícita.
Outra forma de representar dados 3D é por meio da extração de isosuperfícies.
O algoritmo mais conhecido para extração é conhecido como Marching Cubes.
Grande consumo de memória para armazenar o volume resultante
Um voxel é definido pelos valores de seus 8 cantos.
A contribuição do voxel para compor a superfície depende se esses valores são maiores ou menores que um isovalor
Para orientações arbitrárias, cada voxel intersectado pode influenciar o valor de um ou mais pixels no plano.
O pixel resultante pode ser derivado fazendo a re-amostragem dos dados, considerando o voxel mais próximo do plano.    
Rendering volumétrico direto: calcula a cor dos pixels lançando raios no
volume; ou projeta o volume em um plano e usa alguma estratégia para acumular efeitos nos pixels
não são criados polígonos tridimensionais, mas os pixels na imagem resultante são computados individualmente
Mapeamento direto: projeta cada voxel no plano de projeção e determina quais pixels são afetados e de que forma.
Mapeamento inverso(ray casting): lança um raio para cada pixel no plano de projeção através do volume, amostrando valores ao longo desse determinando o valor
de cada pixel.

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

11. Data a técnica para dados multivariados chamada Coordenadas Paralelas:
Explique o layout e funcionamento dessa técnica.
R: Nas coordenadas paralelas, os eixos são paralelos e espaçados uniformemente, e as instâncias de dados são polilinhas que cruzam esses eixos em uma posição proporcional ao valor na dimensão.
Na interpretação desse tipo de visualização podemos:
● Localizar grupos de polilinhas semelhantes (indicam correlação parcial entre pares de dimensões).
● Localizar pontos de cruzamento (indicam correlação parcial negativa).
● Localizar polilinhas diferentes (indicam outliers).

A técnica de Coordenadas Paralelas consiste em um gráfico onde no eixo X estão os atributos e no eixo Y estão o intervalo desses atributos.

b. Cite duas desvantagens dessa técnica e modos pelos quais essas desvantagens podem ser superadas.
R: Interpretar relações entre dimensões não consecutivas pode ser difícil, Interação pode ajudar a sobrepor esse problema indicando relações que se estendem por todas as dimensões.
Grandes grupos sobrepostos: Semi-transparência.

arestas, com as posições dos nós iterativamente refinadas até se atingir um estado de equilíbrio com
o stress do sistema minimizado. Em cada iteração as forças sobre cada nó são calculadas, suas posições atualizadas e suas forças recalculadas.

O pipeline de visualização apresenta diferentes estágios:
Modelagem dos dados; Seleção dos dados; Mapeamento visual dos dados; Definição dos parâmetros de cena; Geração da visualização.

Scatterplots
Uma das visualizações mais antigas desenvolvidas.Dois atributos são comparados mapeando cada instância de dados em um gráfico, podendo outros serem mapeados para cor e tamanho desse.
A visualização pode ser executada para diferentes fins:
Exploração: usuário tem os dados e quer verificar certas características.
Confirmação: existe uma hipótese sobre alguma característica e o usuário quer confirmá-la.
Apresentação: apresentar algum conceito ou conjunto de fatos a um público.


Uma variável pode ser:
Independente (ivi): seu valor não é afetado ou controlado por outra variável.
Exemplo: o tempo em uma série temporal.
Dependente (dvj): é afetada pela variação de uma ou mais variáveis independentes
Exemplo: a temperatura de uma região pode ser dependente da data, horário ou localização.

Oito variáveis visuais:
1 posição; 2 forma (ou marcador); 3 tamanho; 4 brilho; 5 cor; 6 orientação; 7 textura; 8 movimento.

Taxonomia
Uma taxonomia é um meio para transmitir uma classificação. Frequentemente hierárquica, uma taxonomia agrupa objetos similares e define relacionamentos. Ex: uni-dimensional linear; mapa bidimensional; mundo tridimensional; temporal; multidimensional; árvore; rede.

Distorção interativa – Mostrar porções os dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe.
Exemplos: distorções hiperbólicas e distorções esféricas (fisheye lens)

Linking e brushing interativo – Combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Mudanças interativas feitas em uma visualização são automaticamente refletidas nas outras visualizações. Exemplos: Polaris and Scalable Framework.

Confirmação: existe uma hipótese sobre alguma característica e o usuário quer confirmá-la.
Apresentação: apresentar algum conceito ou conjunto de fatos a um público.


Uma variável pode ser:
Independente (ivi): seu valor não é afetado ou controlado por outra variável.
Exemplo: o tempo em uma série temporal.
Dependente (dvj): é afetada pela variação de uma ou mais variáveis independentes
Exemplo: a temperatura de uma região pode ser dependente da data, horário ou localização.

Oito variáveis visuais:
1 posição; 2 forma (ou marcador); 3 tamanho; 4 brilho; 5 cor; 6 orientação; 7 textura; 8 movimento.

Taxonomia
Uma taxonomia é um meio para transmitir uma classificação. Frequentemente hierárquica, uma taxonomia agrupa objetos similares e define relacionamentos. Ex: uni-dimensional linear; mapa bidimensional; mundo tridimensional; temporal; multidimensional; árvore; rede.

Distorção interativa – Mostrar porções os dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe.
Exemplos: distorções hiperbólicas e distorções esféricas (fisheye lens)

Linking e brushing interativo – Combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Mudanças interativas feitas em uma visualização são automaticamente refletidas nas outras visualizações. Exemplos: Polaris and Scalable Framework.

R: Visualização é a comunicação da informação através de representações gráficas. Visualização científica é uma área dedicada à geração de imagens que auxiliam seus usuários na compreensão de dados e processos. Visualização de Informações é uma área de aplicação de técnicas de computação gráfica, geralmente interativas, visando auxiliar o processo de análise e compreensão de um conjunto de dados, através de representações gráficas manipuláveis.

2. Qual a relação da Visualização de Informação com a área de Mineração de Dados?
R: A mineração de dados, têm como objetivo extrair e analisar dados, é nesse último quesito que a visualização de informação entra, isso porque, com a visualização de informações, pode-se transmitir as informações mais rápida e claramente, sendo que podem-se expor os padrões encontrados de forma gráfica, em um modelo onde fique mais intuitivo entender e encontrar informações do que apenas com os dados minerados sem processamento ou visualização.

3. O que são scatterplots? Como eles são criados e como podem ser usados? Qual a principal desvantagem dos scatterplots e como ela pode ser superada?
R: Scatterplots é um tipo de visualização onde os dados são representados em um gráfico no plano cartesiano, através das coordenadas x e y. Portanto, pode ser comparado dois
atributos mapeando cada instância dos dados no plano. É uma ótima forma de visualizar a correlação de duas ou mais medidas ao mesmo tempo
-Desvantagens:  pode ser difícil de ser entendida por um usuário inexperiente,
pois contém valor de medida nos dois eixos, e a terceira medida, que é opcional,
contribui para a complexidade da interpretação.
-Usa rótulos descritivos é uma boa forma de facilitar a interpretação da visualização.
-Os valores podem ser colocados uns sobre os outros e não serão visíveis até que você aplique zoom.

4. Qual a importância do pré-processamento dos dados? Cite e explique dois métodos de pré-processamento de dados.
R: Na maioria dos casos, as bases de dados utilizadas contém informações incompletas(dados faltando), unidades de medidas diferentes, e alguns casos até dados errados. Então, para um melhor aproveitamento é necessário fazer um tratamento nos dados, para que fiquem concisos, tal técnica é chamada de pré-processamento dos dados. Um possível método para tratamento de dados faltantes é o de assinalar um valor sentinela, utilizando a média dos valores, ou a vizinhança mais próxima. Outro método utilizado nas situações em que a dimensionalidade excede a capacidade da técnica de visualização é o de redução, preservando o máximo possível de informação contida nos dados. Essa redução pode ser feita manualmente, selecionando os atributos, ou usando alguma técnica computacional.

6. Qual a importância da interação na visualização de dados?

Ou seja, a interação pode contribuir para mostrar informações referentes a cada parte de uma visualização, também pode mostrar a influência de uma região dela em outra, se bem produzida pode até ser determinante na compreensão de uma visualização. 

7) A taxonomia de Keim classifica as técnicas de interação e distorção em cinco categorias. Cite e explique duas dessas categorias.

3c. Zoom interativo - Uma vez que a quantidade máxima de informação pode ser limitada pela resolução de uma tela, o zoom é uma técnica fundamental para superar esta
limitação. Exemplos: TableLens e IVEE/Spotfire.

8. Qual o objetivo do algoritmo Marching Squares e para que tipo de dados ele pode ser aplicado? Explique em detalhes o algoritmo.
R: O algoritmo Marching Squares visa desenhar linhas entre os valores interpolados ao longo das bordas de uma célula, considerando-se dados sobre o peso dos vértices e um
valor de referência (isovalor).
Vértices de uma célula são anotados com:
    ‘1’ para valores iguais ou maiores que o isovalor;
    ‘0’, caso contrário.
Aplique uma interpolação linear ao longo das bordas da célula para calcular a posição exata do contorno da isolinha.
Recomenda-se utilizar para dados 2D, quando se quer exibir um limite,como por exemplo elevação de temperatura.

 9. Quais são as três categorias de técnicas para visualização volumétrica?
Explique o funcionamento geral das técnicas de cada uma dessas categorias. Quais as vantagens e desvantagens do uso de técnicas de cada uma dessas categorias?
R:    
Tecnicas de fatiamento (slicing): dados são capturados considerando um
plano de recorte 2D alinhado com um dos eixos.
Planos de recorte são usados reduzindo os dados de 3D para 2D
Os dados são recortados com um plano dada uma direção e posição.
Em uma implementação simples a normal do plano coincide com um dos eixos dos dados.
Animação pode ser usada para a visualização de associações entre fatias subsequentes
Técnicas de isosuperfície: gera uma superfície dado um valor e usa técnicas de visualização explícita.
Outra forma de representar dados 3D é por meio da extração de isosuperfícies.
O algoritmo mais conhecido para extração é conhecido como Marching Cubes.
Grande consumo de memória para armazenar o volume resultante
Um voxel é definido pelos valores de seus 8 cantos.
A contribuição do voxel para compor a superfície depende se esses valores são maiores ou menores que um isovalor
Para orientações arbitrárias, cada voxel intersectado pode influenciar o valor de um ou mais pixels no plano.
O pixel resultante pode ser derivado fazendo a re-amostragem dos dados, considerando o voxel mais próximo do plano.    
Rendering volumétrico direto: calcula a cor dos pixels lançando raios no
volume; ou projeta o volume em um plano e usa alguma estratégia para acumular efeitos nos pixels
não são criados polígonos tridimensionais, mas os pixels na imagem resultante são computados individualmente
Mapeamento direto: projeta cada voxel no plano de projeção e determina quais pixels são afetados e de que forma.
Mapeamento inverso(ray casting): lança um raio para cada pixel no plano de projeção através do volume, amostrando valores ao longo desse determinando o valor
de cada pixel.

10. Para dados multivariados podemos utilizar técnicas baseadas em ponto, como técnicas de projeção multidimensional. O que é uma projeção multidimensional? Explique os passos de uma das técnicas de projeção explicadas em aula.
R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

11. Data a técnica para dados multivariados chamada Coordenadas Paralelas:
Explique o layout e funcionamento dessa técnica.
R: Nas coordenadas paralelas, os eixos são paralelos e espaçados uniformemente, e as instâncias de dados são polilinhas que cruzam esses eixos em uma posição proporcional ao valor na dimensão.
Na interpretação desse tipo de visualização podemos:
● Localizar grupos de polilinhas semelhantes (indicam correlação parcial entre pares de dimensões).
● Localizar pontos de cruzamento (indicam correlação parcial negativa).
● Localizar polilinhas diferentes (indicam outliers).

A técnica de Coordenadas Paralelas consiste em um gráfico onde no eixo X estão os atributos e no eixo Y estão o intervalo desses atributos.

b. Cite duas desvantagens dessa técnica e modos pelos quais essas desvantagens podem ser superadas.
R: Interpretar relações entre dimensões não consecutivas pode ser difícil, Interação pode ajudar a sobrepor esse problema indicando relações que se estendem por todas as dimensões.
Grandes grupos sobrepostos: Semi-transparência.

R:  Projeção multidimensional são relações presentes no espaço m-dimensional por p-dimensional. Nais quais,  pontos próximos indicam instâncias relacionadas, pontos distantes indicam objetos não-relacionados.

Passos:
    1- Criar uma matriz de similaridade no espaço m-dimensional (Ds) n×n que contém medidas de similaridade entre cada par de objetos (δ(xi,xj)).
    2- Criar uma matriz com a localização dos pontos projetados no espaço p-dimensional (L) n×p. Posição inicial dos pontos pode ser aleatória ou determinada por alguma técnica
como PCA.
    3- Criar uma matriz de similaridade no espaço p-dimensional (Ls) n×n que contém a similaridade entre todos os pares de pontos projetados em L(d(f (xi), f (xj)))).
    4- Compute o valor do stress, S, que é uma medida das diferenças entre Ds e Ls.
    5- Se o valor de S é suficientemente pequeno, ou não mudou significativamente nas últimas iterações, o algoritmo termina.
    6- Caso contrário, tente ajustar a posição dos pontos em L de forma que a função de stress S seja minimizada.
    6- Retorne ao passo 3.
Como em qualquer processo de otimização, existe o risco de algoritmo parar em mínimos locais que ainda possuem um alto valor de stress (adicionar valores aleatórios para atualização
dos pontos projetados ocasionalmente). Os resultados não são únicos, diferentes localizações iniciais dos pontos levam a resultados muito diferentes.

O pipeline de visualização apresenta diferentes estágios:
Modelagem dos dados; Seleção dos dados; Mapeamento visual dos dados; Definição dos parâmetros de cena; Geração da visualização.

Scatterplots
Uma das visualizações mais antigas desenvolvidas.Dois atributos são comparados mapeando cada instância de dados em um gráfico, podendo outros serem mapeados para cor e tamanho desse.
A visualização pode ser executada para diferentes fins:
Exploração: usuário tem os dados e quer verificar certas características.
Confirmação: existe uma hipótese sobre alguma característica e o usuário quer confirmá-la.
Apresentação: apresentar algum conceito ou conjunto de fatos a um público.


Uma variável pode ser:
Independente (ivi): seu valor não é afetado ou controlado por outra variável.
Exemplo: o tempo em uma série temporal.
Dependente (dvj): é afetada pela variação de uma ou mais variáveis independentes
Exemplo: a temperatura de uma região pode ser dependente da data, horário ou localização.

Oito variáveis visuais:
1 posição; 2 forma (ou marcador); 3 tamanho; 4 brilho; 5 cor; 6 orientação; 7 textura; 8 movimento.

Taxonomia
Uma taxonomia é um meio para transmitir uma classificação. Frequentemente hierárquica, uma taxonomia agrupa objetos similares e define relacionamentos. Ex: uni-dimensional linear; mapa bidimensional; mundo tridimensional; temporal; multidimensional; árvore; rede.

Distorção interativa – Mostrar porções os dados com um alto nível de detalhe, enquanto outros são mostrados com um menor nível de detalhe.
Exemplos: distorções hiperbólicas e distorções esféricas (fisheye lens)

Linking e brushing interativo – Combinar diferentes métodos de visualização para superar as deficiências de técnicas individuais. Mudanças interativas feitas em uma visualização são automaticamente refletidas nas outras visualizações. Exemplos: Polaris and Scalable Framework.
